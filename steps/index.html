Django Forms Tutorial - Step 1

Step 1: Setting Up Your Django Project with MySQL Database

This tutorial will guide you through setting up a Django project and connecting it to a MySQL database.

Commands to Execute

# Install Django and MySQL Libraries
pip install django
pip install mysqlclient

# Create a New Django Project
django-admin startproject form_project
cd form_project

# Create a Django App
python manage.py startapp form_app

Folder Structure After Setup

Configure Database in settings.py

DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.mysql',
        'NAME': 'django_forms_db',
        'USER': 'your_mysql_user',
        'PASSWORD': 'your_mysql_password',
        'HOST': 'localhost',
        'PORT': '3306',
    }
}

Run Migrations

python manage.py migrate

Test the Project

Run the server and test if the project is set up correctly by visiting http://127.0.0.1:8000/

python manage.py runserver

MCQ Questions

1. Which command is used to create a new Django project?

  • a) `django-admin createproject`
  • b) `django-admin startproject`
  • c) `django startproject`
Answer: b) `django-admin startproject`

2. Which Python package is required to connect Django to a MySQL database?

  • a) `mysql-python`
  • b) `mysqlclient`
  • c) `django-mysql`
Answer: b) `mysqlclient`

3. What is the default port for MySQL?

  • a) 8080
  • b) 5432
  • c) 3306
Answer: c) 3306

Essay Question

1. Explain the role of the settings.py file in a Django project. Why is it important to configure the database settings correctly?

The settings.py file is the central configuration file for a Django project. It contains important settings such as the database configuration, allowed hosts, middleware, installed apps, and more. Proper configuration of the database settings is crucial because it ensures that the project can connect to the correct database server and use the right credentials. If the database connection is misconfigured, the project will not be able to store or retrieve data, which is vital for dynamic applications.
